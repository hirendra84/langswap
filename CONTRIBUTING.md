## How to contribute

1. Open an issue describing the change before starting significant work, so we can
   agree on the approach.
2. Keep pull requests focused and reasonably small.
3. Match the surrounding code style. Run the dev tooling (`black`, `flake8`) and
   the test suite (`pytest`) before submitting.
4. Reference any related issue in the PR description.
